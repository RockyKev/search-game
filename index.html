Search and find games

This is a test page with 3 types of Search and Find style games.

Note: This page is using my own Stripey.css design system to deploy quickly, only because it's top of mind. For projects, I'd probably use a industry standard like Bootstrap or Tailwind. This is also Pure JS. Not need for React/Vue/etc.

Using Canvas and Phaser

GOAL: Find the blue flower. It will appear in one of 8 spots. Check out gamescene.js for code. After finding it, the game will restart.

PROS: Lots of flexibility. Transitions are clean. All the code is within Javascript and follows a system.

CONS: Uses Phaser, so implementation is done via JS functions.

NOTE: It's easy on purpose. Imagine if the background was full of flowers or something to increase the challenge. Additionally, canvas does need to check for screen size. It doesn't auto-adjust like responsive design without a event listener, which I didn't add.

Using image Maps

GOAL: Find the person. The clickable area is dark on purpose.

PROS: This is a very clean way to implement. Very clean, no framework.

CONS: It's limited in what you can do. The canvas method allows you to juice up a experience. This does not.
